## 1. ARCHITECTURE

### État général

L'architecture est solide et cohérente. La séparation en couches (`types` → `integrations` → `services` → `hooks` → `components` → `pages`) est respectée. Le lazy loading via `React.lazy` est appliqué sur toutes les pages. React Query gère le cache serveur avec des `staleTime` adaptés. Les Edge Functions Deno sont bien séparées de la logique client.

### Points notables

- La migration `20260424100000_rename_panels_to_chapter_canvases.sql` a renommé la table `panels` en `chapter_canvases`. Le code client utilise désormais `chapter_canvases` partout. Le type `Panel = Tables<"chapter_canvases">` dans `types/index.ts` est un alias sémantique correct.
- `generate-scenario-ai` a migré de Groq (Llama 3.3 70B) vers **Google Gemini Flash** (+ fallback Groq). Le `CLAUDE.md` indique encore "Groq — Llama 3.3 70B" → **à corriger**.
- `callNarraMindUpdate` dans `scenarioAI.ts` est marquée `@deprecated` mais pas supprimée.
- Deux fichiers `use-toast` : `src/hooks/use-toast.ts` (source) et `src/components/ui/use-toast.ts` (re-export). Redondant mais inoffensif.

## 8. PERFORMANCE

### Re-renders
- `ChapterDetail.tsx` (~1700 lignes) centralise un state lourd. Chaque frappe dans un prompt déclenche un re-render du composant parent. Les sous-composants `ColorBlockLayer`, `ImageBlockLayer`, `BubbleLayer` étant extraits, l'impact est atténué, mais les callbacks en props créent encore des re-renders en cascade.

### Requêtes N+1
- `reorderChapters` / `reorderScenarioChapters` : N UPDATE séquentiels → À remplacer par `upsert([...], { onConflict: "id" })`.
- `generate-panel-image` : 3 requêtes DB en cascade (canvas → chapter → project) → À remplacer par une jointure SQL unique.

### Bundle
- `html2canvas` + `JSZip` chargés statiquement dans `ChapterDetail` → ~250 KB inutiles au chargement. À lazy-loader au clic Export.
- `framer-motion` importé dans `Landing.tsx` et `Plans.tsx` (lazy-loadés) → impact limité.

### Images
- PNG générés par FAL.ai servis directement depuis Supabase Storage. Pas de WebP, pas de redimensionnement adaptatif, pas de CDN d'images.
- Sheet images (2560×768 PNG) servies sans lazy loading dans `CharacterViewDialog.tsx`.

### Cache React Query
- `usePanels` : `staleTime: 5 min`. Deux sessions simultanées sur le même chapitre ne voient pas les modifications avant 5 min.

# DreamWeave — Design Context (Impeccable)

> Généré le 2026-05-28. Lu par l'Interface Architect avant toute tâche UI.

---

## Identité produit

- **Produit :** Outil créatif de création de webtoons/mangas assisté par IA
- **Audience :** Créateurs sans compétences d'illustration (14-35 ans, fans manga)
- **Objectif émotionnel :** Empowerment — "Je crée des visuels professionnels en secondes"
- **Voix de marque :** Professionnel mais accessible, créatif, inspirant, jamais générique

---

## Stack technique (design)

- shadcn/ui (Radix UI) — base des composants, toujours restyled
- Tailwind CSS 3.4 — utilities en priorité
- Framer Motion / motion/react — animations
- Mode dark : **par défaut et permanent** (pas de toggle light/dark dans l'app)

---

## Système de couleurs — tokens HSL

```
--lavender:       275°  45%  72%    ← accent principal
--lavender-soft:  275°  30%  92%    ← surfaces claires
--peach:           28°  80%  88%    ← accent secondaire chaud
--peach-deep:      20°  70%  75%    ← hover/actif peach
--mint:           170°  35%  78%    ← accent tertiaire
--cream:           40°  40%  96%    ← texte clair sur dark
--bg-dark:        275°  20%   7%    ← fond principal (jamais noir pur)
```

**Règle absolue :** jamais de couleur hardcodée — toujours via tokens CSS ou classes custom.

## Classes custom DreamWeave

| Classe | Usage |
|--------|-------|
| `.glass` | Glassmorphisme : backdrop-blur(24px) + bordure légère + ombre subtile |
| `.gradient-dream` | Lavande → pêche → menthe (135°, alpha 0.3) — fonds décoratifs |
| `.gradient-primary` | Lavande → pêche-deep (135°, opaque) — boutons CTA et actifs |
| `.text-gradient` | Texte dégradé lavande → pêche |
| `.shadow-dream` | Ombre multi-couches signature |
| `.shadow-glow` | Halo lavande au hover |
| `.bg-content` | Fond crème/pêche (light) ou lavande-dark (dark) |

## Typographie

- **Titres / nav :** Quicksand (`font-display`) — weight 600-700
- **Corps :** Nunito — weight 400-500
- **Base :** 110% (~18px)
- **Jamais** : serif par défaut, Inter comme police principale, system-ui visible

---

## Philosophie design (3 principes)

1. **Glassmorphisme cohérent** — chaque surface = verre, pas flat brutal
2. **Outil créatif professionnel** — pas d'interface générique ou shadcn par défaut
3. **Créateurs en mode "flow"** — pas de friction visuelle, pas d'info overload

---

## Motion

- Bibliothèque : `motion/react` (ex Framer Motion)
- Intensité : **6/10** — fluide et purposeful, jamais cinématique
- Spring physics par défaut : `stiffness: 100, damping: 20`
- `prefers-reduced-motion` : toujours respecté
- Animer uniquement `transform` et `opacity` — jamais `top/left/width/height`

---

## Anti-patterns DreamWeave (interdits absolus)

- **NO** cartes blanches plates — toujours `.glass` ou fond teinté lavande
- **NO** accents chauds isolés (orange, rouge) — rester dans lavande/pêche/mint
- **NO** composants shadcn/ui dans leur état par défaut — toujours restyled
- **NO** Inter comme font principale — Quicksand + Nunito uniquement
- **NO** noir pur `#000000` — bg-dark = `275° 20% 7%`
- **NO** gradients IA génériques (purple glow, mesh gradient bleu-violet) — le projet EST lavande, mais avec intention
- **NO** em-dashes (`—`) dans les textes UI — jamais
- **NO** lorem ipsum, données fictives évidentes, avatars SVG génériques
- **NO** trois cards identiques en colonnes égales — diversifier les layouts
- **NO** animations `window.addEventListener('scroll')` — utiliser Motion useScroll()

---

## Accessibilité

- WCAG AA minimum sur tous les contrastes
- États focus visibles en mode dark
- Navigation clavier sur toutes les interactions editor
- `aria-label` sur les icônes sans texte

---

## Pré-vol obligatoire avant livraison UI

Cocher avant de livrer tout composant :

- [ ] Zéro em-dash (`—`) dans les textes visibles
- [ ] Couleurs via tokens uniquement (pas de hardcode)
- [ ] Composants shadcn restyled avec design system DreamWeave
- [ ] Contrastes WCAG AA vérifiés
- [ ] Animations : transform/opacity uniquement + reduced-motion respecté
- [ ] Mode dark vérifié (pas de texte invisible, pas de fond blanc isolé)
- [ ] États loading / empty / error prévus
- [ ] `tsc --noEmit` 0 erreur

## ✅ Accompli

- **Correction cause racine : événements disparaissent après 30s**
  - `useArianeSidebarNotifs` partageait la query key `["lore-proposals"]` avec une `queryFn` différente (sans `lore_event`) et `staleTime: 30_000`
  - React Query prend le staleTime minimum → refetch auto à 30s avec la mauvaise queryFn → cache écrasé sans events
  - Fix : aligner queryFn (inclure `lore_event`) + `staleTime: Infinity` + `refetchOnWindowFocus: false`
  - Fichier : `src/hooks/useArianeSidebarNotifs.ts`

- **Stabilisation cache proposals (staleTime: Infinity)**
  - Proposals ne se refetchent jamais automatiquement (window focus, remontage composant)
  - Uniquement via `invalidateQueries` explicite (après scan/accept/dismiss)
  - Fix intermédiaire `refetchOnMount: false` retiré car bloquait le fetch initial
  - Fichier : `src/hooks/useArianeLoreProposals.ts`

- **Restauration inconditionnelle des événements après purge**
  - La boucle de restauration était enfermée dans `for (const sc of validatedChapters)` — si 0 chapitres validés (migration absente), aucun event restauré
  - Fix : passe de restauration hors boucle AI pour tous les `savedEventsByChapter` non couverts
  - Fichier : `src/hooks/useArianeLoreProposals.ts`

- **Badges "Existe déjà" / "Ignoré" persistants**
  - Avant : stockés uniquement dans cache React Query `["ariane-forced-ids"]` — effacés à chaque refresh de page
  - Fix : `_ariane_reason` ajouté dans `prefill_data` lors du `triggerForceScan` — stocké en BDD, survit aux refreshes
  - Fichier : `src/hooks/useArianeLoreProposals.ts`, `src/components/project/LoreUniversProposalsSheet.tsx`

- **`runScan` n'invalide que si quelque chose a changé** (`didChange` flag)
  - Évite les refetches inutiles au montage quand la BDD est déjà à jour
  - Fichier : `src/hooks/useArianeLoreProposals.ts`

### ⚠️ Nouveau problème — 92 erreurs TypeScript dans LoreGraphView.tsx

`LoreGraphView.tsx` (2 244 lignes) présente de nombreuses erreurs liées au conflit entre le type natif DOM `Node` et le type `Node` de `@xyflow/react`. Erreurs représentatives :
- `TS2344 : LoreNodeData does not satisfy the constraint 'Node'` — le type custom `LoreNodeData` n'est pas assignable au type `Node` de ReactFlow.
- `TS2698 : Spread types may only be created from object types` — tentatives de spread sur des types `unknown`.
- `TS18046 : 'n.position' is of type 'unknown'` — positions ReactFlow non typées correctement.
- `TS2345 : LoreNodeInsert` manque `chapter_id` requis.

**Impact :** la feature Univers / graphe lore est fonctionnelle à l'exécution (erreurs runtime non bloquantes), mais le build strict TypeScript échouerait. Ce n'est pas un blocant Vercel (`tsc --noEmit` n'est pas dans le pipeline de build), mais c'est une dette à solder avant le lancement public.

### Points forts

- Onboarding progressif bien construit (`ArianeTabTourOverlay`, `ArianeStyleOnboardingCard`, `ArianeJourneyCompleteCard`).
- Design system cohérent : tokens HSL, glassmorphisme, Quicksand + Nunito, motion purposeful (Framer Motion). Aucun em-dash, aucun AI-slop copy identifié.

### Frictions identifiées

- **LoreGraphView** : pas de skeleton/loader pendant le mount ReactFlow — l'onglet Univers peut sembler figé sur de gros graphes. Problème accentué par les 92 erreurs TS qui indiquent une intégration ReactFlow encore fragile.

- **Temps de génération** : 30–90s pour un asset. Toast de notification présent mais pas d'indicateur de durée restante ni de notification navigateur en fin de génération.

- **OG/Twitter meta tags** : `og:image = /favicon.svg`, `twitter:card = summary` — un partage social ou une fiche Product Hunt avec ces méta-tags dégradera massivement le CTR.
- **Waitlist absente** : les visiteurs hésitants n'ont aucun filet de capture.

## 5. 💡 Suggestions d'amélioration

### 🗑️ Suppressions

- **`LoreFriseView.tsx`** — 419 lignes dead code. Interdit par la spec Univers v1 (pas de timeline). Supprimer, allège le bundle.
- **Guard `maxProjects !== null` dans Dashboard** — toujours null. Supprimer ou remplacer par un guard pertinent.
- **`ScenarioVersions` dans `src/types/index.ts`** — soit exposer dans l'UI (historique de versions), soit supprimer les types et la table.

### ⚡ Optimisations

- **Corriger les 92 erreurs TypeScript dans `LoreGraphView.tsx` (P0 qualité)** — le conflit `Node` DOM vs `@xyflow/react` se corrige en renommant le type local : `type LoreFlowNode = Node<LoreNodeData>` et en important explicitement le `Node` de ReactFlow. La correction est ciblée et ne touche pas le canvas éditeur (zone protégée).
- **OG / Twitter meta tags (P0 marketing)** — Créer `/public/og-image.png` (1200×630px), mettre à jour `index.html` : `og:image`, `twitter:card: summary_large_image`, `og:url`. Indispensable avant Product Hunt.
- **Chunking Vite (P1)** — `rollupOptions.output.manualChunks` dans `vite.config.ts` pour isoler ReactFlow et recharts. Améliore LCP.
- **Polices Google Fonts (P1)** — Réduire de 16 à 4 familles dans `<head>` (Quicksand, Nunito, Bangers, Comic Neue). Charger les 12 autres à la demande depuis l'éditeur de bulles uniquement.
- **Barre de progression export (P1)** — Afficher un comptage "Panel X / N" pendant l'export ZIP. Rendre panel par panel avec `requestAnimationFrame`.
- **Skeleton `LoreGraphView` (P2)** — Ajouter un skeleton/loader pendant le mount ReactFlow pour éviter l'impression de freeze sur les gros graphes.
- **`ChapterDetail.tsx` split (P2)** — Extraire `useBubbleManagement` et `useColorBlockManagement`. Cible < 1 500 lignes. Consolider les `useEffect` partageant les mêmes dépendances.

### ➕ Additions

- **Capture email / waitlist sur la landing (P0 avant lancement)** — Formulaire simple : email + CTA "Rejoindre la liste" + compteur de spots. Stack suggérée : Resend + table Supabase `waitlist`. Constituer un pipeline de prospects avant l'annonce publique. Source : expérience Dashtoon et Product Hunt best practices.
- **Page `/status` services AI (P1 Quick)** — Badge ou page simple indiquant la disponibilité de FAL.ai. Réduit le support entrant lors des incidents.
- **Notification fin de génération (P1 Quick)** — Notification navigateur (`Notification API`) ou email "Votre personnage est prêt" pour les 30–90s d'attente.
- **Auto-injection sheet personnage dans les prompts panel (P1 Medium)** — Lors de la génération d'un bloc image de case, proposer d'injecter automatiquement la sheet de l'asset sélectionné comme image de référence. Réduit la friction workflow et exploite mieux le Sheet System.
- **Export multi-format PDF (P2 Medium)** — Les concurrents Anifusion et Dashtoon le proposent. Source : [anifusion.ai features](https://anifusion.ai/) (2026).
- **Partage / lecture publique d'un projet (P3 Long)** — Lien public en lecture seule. Différenciateur vs Jenova et MangaBuilder, vecteur d'acquisition organique.

## 4. Services & Hooks

### P1 — `canGenerate()` ne vérifie pas le quota mensuel
- **Fichier** : `src/hooks/useAssetGeneration.ts`
- **Description** : La fonction vérifie le style template et le plan, mais pas `usageInfo.count < usageInfo.limit`. Un utilisateur Free à 20/20 peut toujours appeler FAL (l'Edge Function le bloque, mais le bouton est toujours actif).
- **Fix** :
  ```ts
  const canGenerate = useCallback(() => {
    if (usageInfo.count >= usageInfo.limit) return false;
    // … reste des vérifications
  }, [usageInfo]);
  ```

### P2 — `fetchProjects()` repose uniquement sur RLS Supabase
- **Fichier** : `src/services/projects.ts`
- **Description** : Pas de `.eq("user_id", userId)` côté client. Si la RLS est mal configurée sur un nouvel environnement, l'utilisateur voit tous les projets.
- **Fix** : Ajouter `.eq("user_id", userId)` en garde defensive.

### P2 — `useCallback` manquant sur `handleNavigateToCreateAsset`
- **Fichier** : `src/pages/ProjectDetail.tsx` ligne 65
- **Statut** : ✅ Déjà wrappé avec `useCallback` dans le fichier actuel.

### P3 — `callBaselineAI` exporté mais plus utilisé
- **Fichier** : `src/services/scenarioAI.ts`

## 5. Sécurité & RLS

### P0 — FK cassée memory_entities/memory_summaries → profiles(id) au lieu de auth.users(id)
- **Fichier** : `supabase/migrations/20260423200000_fix_narramind_fk.sql`
- **Description** : Les inserts NarraMind échouaient silencieusement (contrainte FK violée). Migration correctrice créée.
- **Statut** : ⚠️ Migration à appliquer manuellement dans Supabase SQL Editor.
- **Vérification post-apply** :
  ```sql
  SELECT COUNT(*) FROM memory_entities WHERE user_id NOT IN (SELECT id FROM auth.users);
  ```

### P1 — Fallback `SUPABASE_ANON_KEY || serviceKey` dans create-checkout-session
- **Fichier** : `supabase/functions/create-checkout-session/index.ts` ligne ~63
- **Description** : Si `SUPABASE_ANON_KEY` n'est pas définie, la vérification JWT utilise la `serviceKey` comme apikey, ce qui n'est pas la clé attendue par `/auth/v1/user`.
- **Fix** : Rejeter explicitement si `SUPABASE_ANON_KEY` est absent :
  ```ts
  const anonKey = Deno.env.get("SUPABASE_ANON_KEY");
  if (!anonKey) return jsonResponse({ error: "SUPABASE_ANON_KEY manquante." }, 500);
  ```

### P2 — RLS `memory_entities`/`memory_summaries` sans séparation SELECT/INSERT
- **Fichier** : `supabase/migrations/20260423120000_add_narramind_tables.sql`
- **Description** : Policy `FOR ALL USING (auth.uid() = user_id)` couvre toutes les opérations. Standard acceptable mais insuffisant pour prévenir les attaques par confusion de rôle.
- **Fix** : Séparer en policies distinctes SELECT / INSERT / UPDATE / DELETE avec WITH CHECK.

## 6. Edge Functions

### P1 — generate-scenario-ai : fallback vers modèle dégradé silencieux
- **Fichier** : `supabase/functions/generate-scenario-ai/index.ts`
- **Description** : Sur 429 Gemini, bascule vers un modèle plus faible sans en informer le client. L'utilisateur peut recevoir une génération de moindre qualité sans le savoir.
- **Fix** : Retourner un champ `"model_used": "fallback"` dans la réponse JSON. Côté client, afficher un toast informatif si `model_used === "fallback"`.

### P2 — generate-asset-image : pas de retry sur 429 FAL
- **Fichier** : `supabase/functions/generate-asset-image/index.ts`
- **Description** : Si FAL retourne 429 (rate limit), l'erreur remonte directement à l'utilisateur sans retry.
- **Fix** : Implémenter un retry exponentiel (1-2 tentatives, délai 2s/5s) avant de renvoyer l'erreur.

### P3 — Tous les Edge Functions : `verify_jwt = false` dans config.toml
- **Fichier** : `supabase/config.toml`
- **Description** : Correctement configuré pour ES256. Chaque fonction appelle `verifyUserFromToken()` manuellement. ✅

## 7. Performance

### P2 — `transition-all` dans composants shadcn/ui
- **Fichiers** : `src/components/ui/accordion.tsx`, `tabs.tsx`, `toast.tsx`, `input-otp.tsx`
- **Description** : Ces classes forcent le recalcul layout à chaque frame d'animation.

### P2 — Hero background non optimisé
- **Fichier** : `src/pages/Landing.tsx`
- **Description** : `hero-bg.jpg` chargé via CSS backgroundImage. Pas de préchargement, pas de WebP.
- **Fix** : Ajouter `<link rel="preload" as="image" href="/hero-bg.jpg">` dans le `<head>`, ou convertir en WebP.

### P2 — `ImageWithFallback` — `lazy=false` par défaut
- **Fichier** : `src/components/ui/ImageWithFallback.tsx` ligne 23
- **Description** : Toutes les images utilisant ce composant sans spécifier `lazy` chargent en mode eager.
- **Fix** : Changer le défaut en `lazy = true`.

### P3 — Pas de srcset / WebP pour les images assets
- **Description** : Les URLs FAL sont des JPEG/PNG pleine résolution, servies identiques sur mobile et desktop.
- **Fix** : Utiliser un CDN image (ex: Cloudflare Images, Imgproxy) pour redimensionner à la volée.
